# Contributing to Gratis

## Before you start

Check the issue tracker before opening a new issue or starting work on a feature. If you want to build something significant, open an issue first so we can discuss it before you invest the time.

## Pull requests

- Keep PRs focused — one thing per PR
- If you're fixing a bug, include a description of what caused it
- If you're adding a feature, explain why it belongs in Gratis rather than as a separate tool
- All code runs on the server. Security is not optional — think about what you're exposing and to whom

## Code style

Enforced by tooling where possible. Run the linter before submitting.
